VoiceAttack: variables

A guide on how to work with variables within VoiceAttack

The default syntax for a variable is: EliteAPI.(Name). For example, the LowFuel variable provided by EliteVA would be EliteAPI.LowFuel.

[2]Using variables in conditions

VoiceAttack provides a basic conditional statements system with conditional statements. If we'd want to execute a specific action only when we are low on fuel, we could check the variable for a specific value. For example, our LowFuel variable. If we'd want to check whether we're low on fuel, we can use the following if statement: EliteAPI.LowFuel = "True". This will execute the actions below the if statement if we are low on fuel, and execute everything below the else statement if we are not.

[*]A list of all available variables

EliteVA provides a number of constantly updated variables that can give you a sense of what's going on in the game. You can find a list of them below.

Variable Explanation
EliteAPI.Docked This will return true when the player is docked at a station.
EliteAPI.Landed This will return true when the player is currently landed on a landable planet.
EliteAPI.Gear This token will return true whenever the landing gear has been lowered.
EliteAPI.Shields Returns true only if the ship's shields are online. Note that this is only true or false, and not a percentage.
EliteAPI.Supercruise Returns true when in supercruise.
EliteAPI.FlightAssist Will return true if the ship has flight assist enabled.
EliteAPI.Hardpoints Returns true if the ship's hardpoints (or weapons) are deployed.
EliteAPI.Winging If the player is currently in a wing, this token will return true.
EliteAPI.Lights When the ship's lights are turned on, this will return true.
EliteAPI.CargoScoop Returns true whenever the cargo scoop is open.
EliteAPI.SilentRunning If the Commander has silent running enabled, this will return true.
EliteAPI.Scooping Returns true whenever the ship is scooping fuel from a star.
EliteAPI.SrvHandbreak This token returns true whenever the handbreak on the SRV is being used.
EliteAPI.SrvTurrent If the turrent is being controlled by the player, this token will return true.
EliteAPI.SrvNearShip Returns true when the SRV turrent cannot be deployed, thus the SRV being close to the ship.
EliteAPI.SrvDriveAssist Returns true whenever the player has drive assist enabled in their SRV.
EliteAPI.MassLocked If the ship is mass locked, this will return true.
EliteAPI.FsdCharging If the frameshiftdrive is charging up for a jump, this will return true.
EliteAPI.FsdCooldow Returns true when the frameshiftdrive is cooling down.
EliteAPI.LowFuel If the ship is low on fuel (<25%), this token will say true.
EliteAPI.Overheating If the ship's overheating this will return true.
EliteAPI.HasLatLong Will return true when latitude and longitude on a planet are available.
EliteAPI.InDanger Returns true if the player is in combat/danger.
EliteAPI.InInterdiction If the player is being interdicted, this will return true. It will not return true in the case of the player interdicting another ship.
EliteAPI.InMothership If the player is in the mothership, and thus not in a fighter/srv, this will return true.
EliteAPI.InNoFireZone True if the ship is currently in a no fire zone.
EliteAPI.InFighter Returns true when the player is in a fighter.
EliteAPI.InSRV Returns true when the player is in a SRV.
EliteAPI.AnalysisMode Returns true if the ship's current mode is in analysis mode, and thus not in combat mode.
EliteAPI.NightVision Returns true when night vision is enabled. (Currently bugged, Frontier is working on it)
EliteAPI.Pips.Systems Returns a number [0-8] for the amount of pips designated for systems/shields.
EliteAPI.Pips.Engines Returns a number [0-8] for the amount of pips designated for the engines.
EliteAPI.Pips.Weapons Returns a number [0-8] for the amount of pips designated for weapons/hardpoints.
EliteAPI.Firegroup Returns the current firegroup.
EliteAPI.GuiFocus Returns the currently focussed panel.
EliteAPI.Fuel The amount of fuel the ship currently has.
EliteAPI.FuelReservoir The amount of fuel the ship currently has in its reservoir.
EliteAPI.FuelMax The maximum amount of fuel the ship can hold.
EliteAPI.Cargo The number of tonnes of cargo currently being transported by the ship (EliteAPI.Cargo in 2.0.0.0).
EliteAPI.Commander Returns the name of the commander.
EliteAPI.System The name of the last known (often current) system.
EliteAPI.Body The name of the last know body, this could be a station or a planet.
EliteAPI.BodyTyope The type of the last known body, could be planet, station, rings, etc.
EliteAPI.Rank.Combat Returns the combat rank (as a the title).
EliteAPI.Rank.Cqc Returns the CQC rank (as a number).
EliteAPI.Rank.Trade Returns the trade rank (as a the title).
EliteAPI.Rank.Exploration Returns the exploration rank (as a the title).
