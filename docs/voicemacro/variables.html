Voicemacro: variables

A guide on how to work with variables within VoiceMacro

Every VoiceMacro variable must have a suffix

Booleans are represented as 'True' and 'False', mind the capitalisation

[1]The syntax of variables

Every variable in VoiceMacro has a prefix. Different prefixes mean different things. There are 5 different types of variables in VoiceMacro.

Type Suffix Meaning
Internal None Used by VoiceMacro to hold variables such as the current time and other read-only information.
Local _l These variables cannot be accessed outside of the macro.
Profile _p These variables can be accessed by every macro, given they are in the same profile.
Global _g These variables can be accessed by every macro and every profile.
Global (S) _s These variables can be accessed by every macro and every profile and are saved to the disk.

EliteMacro only uses profile variables, therefore every EliteMacro variable will have the _p suffix. Any green field in VoiceMacro can use variables. The default syntax for a variable is: {EliteAPI.(Name)(Suffix)}. For example, the LowFuel variable provided by EliteMacro would be {EliteAPI.LowFuel_p}.

[2]Using variables in conditions

VoiceMacro provides a basic conditional statements system with if and else statements. If we'd want to execute a specific macro only when we are low on fuel, we could check the variable for a specific value. For example, our LowFuel variable. If we'd want to check whether we're low on fuel, we can use the following if statement: EliteAPI.LowFuel_p = "True". This will execute the macros below the if statement if we are low on fuel, and execute everything below the else statement if we are not.

Action Value
Condition EliteAPI.LowFuel_p = "True"
WriteToLog We are low on fuel!
Condition Else
WriteToLog We are not low on fuel!
Condition EndIf
[*]A list of all available variables

EliteMacro provides a number of constantly updated variables that can give you a sense of what's going on in the game. You can find a list of them below.

Variable Explanation
{EliteAPI.Docked_p} This will return true when the player is docked at a station.
{EliteAPI.Landed_p} This will return true when the player is currently landed on a landable planet.
{EliteAPI.Gear_p} This token will return true whenever the landing gear has been lowered.
{EliteAPI.Shields_p} Returns true only if the ship's shields are online. Note that this is only true or false, and not a percentage.
{EliteAPI.Supercruise_p} Returns true when in supercruise.
{EliteAPI.FlightAssist_p} Will return true if the ship has flight assist enabled.
{EliteAPI.Hardpoints_p} Returns true if the ship's hardpoints (or weapons) are deployed.
{EliteAPI.Winging_p} If the player is currently in a wing, this token will return true.
{EliteAPI.Lights_p} When the ship's lights are turned on, this will return true.
{EliteAPI.CargoScoop_p} Returns true whenever the cargo scoop is open.
{EliteAPI.SilentRunning_p} If the Commander has silent running enabled, this will return true.
{EliteAPI.Scooping_p} Returns true whenever the ship is scooping fuel from a star.
{EliteAPI.SrvHandbreak_p} This token returns true whenever the handbreak on the SRV is being used.
{EliteAPI.SrvTurrent_p} If the turrent is being controlled by the player, this token will return true.
{EliteAPI.SrvNearShip_p} Returns true when the SRV turrent cannot be deployed, thus the SRV being close to the ship.
{EliteAPI.SrvDriveAssist_p} Returns true whenever the player has drive assist enabled in their SRV.
{EliteAPI.MassLocked_p} If the ship is mass locked, this will return true.
{EliteAPI.FsdCharging_p} If the frameshiftdrive is charging up for a jump, this will return true.
{EliteAPI.FsdCooldow_p} Returns true when the frameshiftdrive is cooling down.
{EliteAPI.LowFuel_p} If the ship is low on fuel (<25%), this token will say true.
{EliteAPI.Overheating_p} If the ship's overheating this will return true.
{EliteAPI.HasLatLong_p} Will return true when latitude and longitude on a planet are available.
{EliteAPI.InDanger_p} Returns true if the player is in combat/danger.
{EliteAPI.InInterdiction_p} If the player is being interdicted, this will return true. It will not return true in the case of the player interdicting another ship.
{EliteAPI.InMothership_p} If the player is in the mothership, and thus not in a fighter/srv, this will return true.
{EliteAPI.InNoFireZone_p} True if the ship is currently in a no fire zone.
{EliteAPI.InFighter_p} Returns true when the player is in a fighter.
{EliteAPI.InSRV_p} Returns true when the player is in a SRV.
{EliteAPI.AnalysisMode_p} Returns true if the ship's current mode is in analysis mode, and thus not in combat mode.
{EliteAPI.NightVision_p} Returns true when night vision is enabled. (Currently bugged, Frontier is working on it)
{EliteAPI.Pips.Systems_p} Returns a number [0-8] for the amount of pips designated for systems/shields.
{EliteAPI.Pips.Engines_p} Returns a number [0-8] for the amount of pips designated for the engines.
{EliteAPI.Pips.Weapons_p} Returns a number [0-8] for the amount of pips designated for weapons/hardpoints.
{EliteAPI.Firegroup_p} Returns the current firegroup.
{EliteAPI.GuiFocus_p} Returns the currently focussed panel.
{EliteAPI.Fuel_p} The amount of fuel the ship currently has.
{EliteAPI.FuelReservoir_p} The amount of fuel the ship currently has in its reservoir.
{EliteAPI.FuelMax_p} The maximum amount of fuel the ship can hold.
{EliteAPI.Cargo_p} The number of tonnes of cargo currently being transported by the ship (EliteAPI.Cargo in 2.0.0.0).
{EliteAPI.Commander_p} Returns the name of the commander.
{EliteAPI.System_p} The name of the last known (often current) system.
{EliteAPI.Body_p} The name of the last know body, this could be a station or a planet.
{EliteAPI.BodyTyope_p} The type of the last known body, could be planet, station, rings, etc.
{EliteAPI.Rank.Combat_p} Returns the combat rank (as a the title).
{EliteAPI.Rank.Cqc_p} Returns the CQC rank (as a number).
{EliteAPI.Rank.Trade_p} Returns the trade rank (as a the title).
{EliteAPI.Rank.Exploration_p} Returns the exploration rank (as a the title).
